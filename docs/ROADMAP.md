# 演进路线

当前重点是 **V0-A：固定环境来电自动接听可行性门禁**。先跑通一台手机和一个微信版本，再扩大兼容范围；V0 不引入 Agent 推理。

## V0-A：来电自动接听

固定目标：`HUAWEI Pura 70 Ultra / HarmonyOS 4.2.0 / WeChat 8.0.76`。

- 使用家属维护的唯一微信备注做本地白名单键，完整文本精确匹配，缺失或歧义时默认拒绝。
- 在所有自动化服务关闭时采集六组只读快照：语音/视频 x 未锁屏/锁屏亮屏/锁屏熄屏。
- 快照必须同时出现微信包名、白名单备注、通话类型、精确接听标签和可点击目标，且不存在高风险关键词。
- 快照门禁和人工截图复核通过后，才启用 GrandmaBridge 做负向测试和六组真实来电测试。
- 本地日志记录识别、白名单、安全判定和点击结果，不记录聊天内容。

V0-A 通过条件是六种状态均能稳定接听白名单来电，非白名单和非通话页面始终不点击。未锁屏单点成功只能记录为部分可行，不能通过 V0-A。

## V0.5：一键拨出

- 仅由用户在 GrandmaBridge 内主动触发。
- 仅允许本地白名单联系人和语音/视频通话。
- 从已确认的微信主标签页开始，逐步精确确认搜索页、联系人和目标聊天页。
- 任意页面歧义、超时、高风险关键词或来电插入都立即停止。

V0.5 不阻塞 V0-A。完成 V0-A 后再单独验证，且全程人工观察。

## V1：受控 Agent 化

- 云端或本地 `GrandmaAgentServer` 管理配置、设备状态、审计和窄类型工具请求。
- GrandmaBridge 只暴露 `answer_wechat_call`、`make_wechat_voice_call`、`make_wechat_video_call` 等明确通话工具，不提供通用点击、输入或任意 UI 工具。
- 实时自动接听走确定性本地路径，不把 LLM 放在时延敏感的接听决策链中。
- 云端 `SafetyGate` 可提前拒绝；手机本地 `SafetyGate` 保留不可绕过的最终否决权。
- 设备上报权限、服务状态和心跳，不上传聊天内容。

## V2：语音交互

- 外婆说“找思怡”。
- ASR 识别并映射到家属确认的联系人。
- Agent 只选择受控通话工具；歧义时询问，不做模糊联系人匹配。

## V3：完整照护作品

- 家属控制台、远程状态、未接通提醒和权限安全说明。
- 在新的安全评审和独立授权后，再考虑吃药提醒、求助和陪伴。

## 开发纪律

每个版本都必须保持窄动作空间、默认拒绝、明确白名单或人工确认、可回退和可审计。微信 UI、系统版本或设备型号变化后，原门禁自动失效并重新采集快照；不得通过放宽选择器“修复”未知页面。
