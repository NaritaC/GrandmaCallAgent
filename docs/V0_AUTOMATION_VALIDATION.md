# V0 自动化验证计划

V0-A 是固定环境的可行性门禁，不是 Agent 功能验收。它只验证微信白名单语音/视频来电能否由手机本地逻辑安全接听。

## 固定边界

| 项目 | V0-A 固定值 |
| --- | --- |
| 接听设备 | HUAWEI Pura 70 Ultra |
| 系统 | HarmonyOS 4.2.0 |
| 微信 | 8.0.76 |
| 白名单键 | 家属维护的唯一微信备注，精确匹配 |
| 网络/Agent | 不依赖 |

呼叫方可以使用 Android 或 iPhone。微信号仅作辅助记录，因为来电 Accessibility UI 未证明能稳定暴露微信号。

## 分阶段门禁

### Gate 0：离线与构建

- `scripts/v0_self_test.ps1` 通过。
- Android 单元测试、Lint 和 debug APK 构建通过。
- 自动接听开关保持关闭。

### Gate 1：只读界面快照

关闭 GrandmaBridge、GKD 等所有 Accessibility 自动化服务，使用测试微信号和合成备注 `V0TEST01` 采集：

1. 未锁屏语音、未锁屏视频。
2. 锁屏亮屏语音、锁屏亮屏视频。
3. 锁屏熄屏语音、锁屏熄屏视频。

每份 XML 必须同时满足：活动根节点属于 `com.tencent.mm`、目标备注可见、通话类型可见、精确接听标签可见且能定位到启用的可点击祖先、页面无支付/转账/删除等风险词。测试人还必须人工检查对应截图。任一项失败即停止，不创建宽泛规则，不启用自动化。

### Gate 2：负向实测

- 总开关关闭时不接听。
- 非白名单来电不接听。
- 非通话页面出现“接受/同意”时不点击。
- 包名、联系人、通话类型或页面状态不明确时默认拒绝。

### Gate 3：白名单实测

在直接监督、备用呼叫方和测试账号条件下，对 Gate 1 的六种组合逐一运行 GrandmaBridge。每次必须出现 `incoming_detected`、`incoming_allowed` 和 `accept_success`；视频还必须记录 `callType=video`。

## V0-A 完成标准

- 六组只读快照门禁通过并人工复核。
- 三类负向测试均无点击。
- 六组白名单真实来电均成功接听，重复测试无跨页面误触。
- 日志能解释每次允许、拒绝和失败原因。
- 无任何支付、转账、红包、删除、发消息、好友管理或聊天读取能力。

如果锁屏熄屏时 UI 根节点属于系统界面、联系人不可见或接听节点不可访问，应记录为平台限制。不得伪造通过结果；V0-A 保持未通过，再评估通知全屏 Intent、系统设置或不同运行方案。

## V0.5 单独验收

一键拨出是多页面状态机，误触面显著高于接听，因此移到 V0.5。只有 V0-A 通过后，才运行 `OutboundWrongPage`、`HighRiskPage`、`OutboundCancel`、`OutboundVoice` 和 `OutboundVideo`，并显式传入实验开关。

具体命令和应急停止方式见 [V0 手机验证指南](V0_PHONE_VALIDATION.md)。GKD 只作为独立校准工具，见 [GKD 校准说明](V0_GKD_VALIDATION.md)。
