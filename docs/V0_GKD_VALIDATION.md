# GKD 校准说明

GKD 用作 V0-A 的独立快照和规则验证工具，不作为 GrandmaBridge 的生产依赖。它可以快速证明某个真实微信来电节点能否被稳定匹配，但不能证明长期安全或兼容所有版本。

## 使用边界

- 只使用 [GKD 官方仓库](https://github.com/gkd-kit/gkd) 和官方文档。
- GKD 采用 GPL-3.0，README 另有非商业使用说明；未完成许可证评审前，不复制源码、不打包进 GrandmaBridge、不分发衍生 APK。
- 带网银的手机不建议安装额外自动化工具。可直接使用本仓库的只读 ADB 快照；GKD 校准优先放在无敏感 App 的备用机。
- GKD 和 GrandmaBridge 无障碍服务不能同时启用。

## 校准顺序

1. 先按 [手机验证指南](V0_PHONE_VALIDATION.md) 完成六组只读快照。
2. 从真实快照确认微信包名、版本、Activity、联系人备注节点、通话类型节点和可点击接听节点。
3. 使用 GKD 的本地订阅功能创建最小规则。没有真实节点证据前不填写 selector，不猜 `resource-id`。
4. 规则必须同时约束目标 App/版本/Activity、精确联系人备注、语音或视频信号和接听目标；高风险文本作为排除条件。
5. 将单次动作上限设为 1，规则默认关闭。每次只启用一条规则并由测试人在旁观察。
6. 按未锁屏、锁屏亮屏、锁屏熄屏分别测试语音和视频；每轮结束立即关闭 GKD 无障碍服务。

具体 selector、`matches`、`excludeMatches`、作用域和快照操作以 [订阅指南](https://gkd.li/guide/subscription)、[选择器指南](https://gkd.li/guide/selector) 和 [快照指南](https://gkd.li/guide/snapshot) 为准。

## 通过条件

- 规则只在目标微信来电页面命中。
- 联系人或通话类型不一致时不命中。
- 非通话“接受”按钮和含风险词页面不命中。
- 每次来电最多执行一次接听动作。
- 六种屏幕/通话组合都有独立记录。

任一条件失败就停用规则并保留快照，不通过增加通配符、删除联系人条件或放宽到包含匹配来追求成功。校准得到的真实节点信号应回填 GrandmaBridge 的确定性解析器和测试，而不是在老人手机上长期同时运行两套自动化。
