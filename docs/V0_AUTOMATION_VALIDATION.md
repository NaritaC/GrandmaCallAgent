# V0 自动化验证计划

V0 的目标不是实现 Agent 能力，而是在真实手机上验证微信通话自动化是否可行。当前阶段只写和验证本地自动化脚本/原型，不依赖云端 Agent Server、工具 API、设备心跳或云端决策。

## 验证目标

- 自动识别微信语音/视频来电。
- 在本地完成白名单判断。
- 对白名单联系人自动接听。
- 支持一键拨出微信语音/视频通话。
- 记录本地日志，便于排查识别、判断和点击过程。

## 非目标

- 不接入云端或本地 Agent Server。
- 不暴露 `answer_wechat_call` / `make_wechat_video_call` 工具 API。
- 不做自然语言意图识别、ASR 或对话能力。
- 不做设备状态心跳上报闭环。
- 不执行支付、转账、红包、删除消息、发送消息、加好友、拉群等非通话动作。

## 建议实现路径

1. 先用真机和目标微信版本确认来电页、接听按钮、一键拨出入口的 UI 文本、`resource-id`、`content-desc` 和窗口层级。
2. 写最小本地自动化脚本或 Android 无障碍原型，只覆盖单个动作链路。
3. 每个动作都输出本地日志：当前包名、页面文本摘要、匹配结果、白名单结果、点击目标、执行结果。
4. 先验证白名单联系人来电接听，再验证非白名单不接听。
5. 最后验证一键拨出，并保持拨出动作必须由用户主动点击本地按钮触发。

## 验收清单

- [ ] 微信语音来电能被识别。
- [ ] 微信视频来电能被识别。
- [ ] 白名单联系人来电能自动接听。
- [ ] 非白名单联系人来电不会自动接听。
- [ ] 当前页面不是微信来电页时不会点击。
- [ ] 页面出现支付、转账、红包、删除等关键词时不会点击。
- [ ] 一键拨出能打开指定白名单联系人的微信语音/视频通话入口。
- [ ] 一键拨出只从已确认的微信主标签页开始，并对联系人结果做精确匹配。
- [ ] 微信停在错误页面时不会向任意输入框写入联系人，也不会点击联系人或通话按钮。
- [ ] 本地日志能说明每次允许、拒绝或失败的原因。

## V0 完成标准

V0 完成的标志是真机证明这些自动化动作可以稳定、可解释、可回退地执行。只有 V0 通过后，才进入 V1 的 Agent Server、Bridge 工具 API、Safety Gate 服务化和设备状态上报。

手机实测步骤见 [V0 手机验证指南](V0_PHONE_VALIDATION.md)。
