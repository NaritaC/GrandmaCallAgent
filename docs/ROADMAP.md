# 演进路线

本文记录 GrandmaCallAgent 的产品演进路线。当前实现重点是 **V0：本地自动化脚本验证**。V0 不涉及 Agent 能力；仓库中已有的 FastAPI 服务端和 WebSocket Bridge 只作为 V1 骨架保留。

## V0：本地自动化脚本验证

目标：写出并在真实手机上验证微信通话自动化脚本/原型是否可行，不依赖云端或本地 Agent 推理。

- 自动识别微信来电。
- 白名单判断。
- 自动接听。
- 一键拨出。
- 本地日志。

验收重点：

- 在目标 Android 真机和目标微信版本上可稳定工作。
- 不点击支付、转账、红包、删除消息、发送消息、好友管理等非通话动作。
- 本地日志足够排查识别失败、白名单拒绝和点击失败。
- V0 只验证本地自动化链路，不暴露 Agent 工具 API，不做设备心跳闭环。

## V1：Agent 化

目标：把受控手机能力暴露给云端或本地 Agent Server。

- 云端/本地 Agent Server。
- 手机端 Bridge 暴露工具 API。
- Agent 调用 `answer_wechat_call` / `make_wechat_video_call`。
- Safety Gate。
- 设备状态上报。

## V2：语音交互

目标：让外婆用自然语音发起通话。

- 外婆说“找思怡”。
- ASR 识别。
- Agent 判断意图。
- 调用拨打工具。

## V3：AI 向善完整作品

目标：形成对家属和老人都可信、可用的完整照护产品。

- 家属控制台。
- 远程状态查看。
- 未接通提醒。
- 权限安全说明。
- 可扩展到吃药提醒、求助、陪伴。

## 开发规则

V0 真机稳定前，不引入 Agent 能力，也不扩展泛化 UI 自动化。每个版本都必须保持同一条安全底线：动作范围窄、白名单或人工确认明确、日志可验证。
