# 权限说明

## V0-A Android 权限

| 权限/服务 | 用途 | 当前控制 |
| --- | --- | --- |
| `AccessibilityService` | 读取微信来电节点并点击接听 | XML 用 `android:packageNames="com.tencent.mm"` 限定事件；代码再次检查包名；每次点击经过本地 `SafetyGate` |
| `NotificationListenerService` | 从微信通知识别联系人和通话类型 | Android 授权范围是全部通知，但代码收到事件后立即拒绝非 `com.tencent.mm` 包；不持久化其它通知 |
| `TextToSpeech` | 播报接听或拒绝结果 | 不需要额外危险权限 |

V0-A Manifest **不声明任何 `uses-permission`**，本地 Runtime 不启动 WebSocket。CI 使用 AAPT2 审计最终 APK，防止依赖或 Manifest 合并意外引入权限。V1 若恢复联网能力，必须单独评审权限、认证、`wss://`、数据最小化和本地最终否决机制。

## 用户授权

HarmonyOS/Android 不允许 App 静默打开以下服务，测试人必须在系统设置中手动授权：

- 无障碍服务：`GrandmaBridge`
- 通知使用权：`GrandmaBridge`

无障碍和通知使用权都是高敏感系统权限。即使代码限制微信，也建议在不安装网银和支付工具的备用机首测；每轮测试后立即关闭授权，不要长期留在生产老人手机上。

USB 调试只用于安装、只读快照和读取本地日志。验证结束后应撤销电脑调试授权并关闭 USB 调试。

## V0-A 本地数据

- 白名单：保存在 App 私有目录，只使用家属维护的唯一微信备注。
- 动作日志：记录时间、动作、允许/拒绝和失败原因，可能包含联系人备注。
- 快照：`screen.png` 和 `ui-dump.xml` 可能包含可见私密文本，只保存在已忽略的 `artifacts/` 目录。

V0-A 不上传聊天记录、联系人通讯录、音频、视频、截图、UI XML或设备心跳。

## 永久禁止动作

- 支付、付款、转账、收款、红包、银行卡和验证码操作。
- 删除、撤回、清空聊天、发送消息、添加好友或修改账号。
- 读取或抓取聊天内容、通讯录和其它 App 数据。
- 通用点击、通用输入或任意 UI 自动化工具。

## V1 预留要求

- 只暴露窄类型通话工具，云端不能直接下发坐标、选择器、文本输入或任意动作。
- 每台设备使用独立凭据，WebSocket 只允许受信任的 `wss://` 地址。
- 云端只管理配置、状态和审计；实时自动接听保持确定性本地路径。
- 手机本地 `SafetyGate` 对所有请求保留不可绕过的最终否决权。
